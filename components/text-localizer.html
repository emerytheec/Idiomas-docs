TextLocalizer

Localiza un texto individual con parametros dinamicos, rich text y pluralizacion

Descripcion

TextLocalizer localiza un texto individual. A diferencia de CanvasLocalizer (que gestiona un Canvas completo), TextLocalizer te da control fino sobre un solo texto con funcionalidades avanzadas:

  • Parametros dinamicos: {0}, {1}, {2}
  • Prefijos y sufijos personalizados
  • Formato rich text con {t}
  • Cambio de clave en runtime

Ideal para textos que necesitan datos dinamicos como nombres de jugador, contadores, o mensajes que cambian segun el contexto.

Cuando usar TextLocalizer vs CanvasLocalizer

CaracteristicaCanvasLocalizerTextLocalizer
Tipo de textos Estaticos (se traducen tal cual) Dinamicos (con parametros, rich text)
Cantidad Muchos textos en un Canvas Un texto por componente
Setup Rapido (escaneo automatico) Manual (uno por uno)
Parametros No Si — {0}, {1}, {2}
Rich text No Si — {t} placeholder
Clave dinamica No Si — SetTranslationKey()
ⓘ Puedes usar ambos en el mismo Canvas. CanvasLocalizer ignora automaticamente los textos que ya tienen un TextLocalizer.

Configuracion

PropiedadTipoDescripcion
manager LocalizationManager Referencia al manager. Se asigna automaticamente si hay un solo LocalizationManager en la escena.
translationKey string Clave de traduccion en el JSON (ej: "btn_start", "welcome_message").
prefix string Texto que se añade antes de la traduccion (ej: ">> ").
suffix string Texto que se añade despues de la traduccion (ej: " <<").
richTextFormat string Formato rich text. Usa {t} para la traduccion (ej: "<b>{t}</b>").
uiText Text Componente Legacy Text. Se auto-detecta si existe en el GameObject.
tmpText TextMeshProUGUI Componente TMP. Se auto-detecta si existe en el GameObject.

Parametros dinamicos

Puedes insertar datos dinamicos en las traducciones usando placeholders {0}, {1}, {2}:

JSON:

{
  "en": { "welcome": "Hello {0}, you have {1} messages" },
  "es": { "welcome": "Hola {0}, tienes {1} mensajes" },
  "ja": { "welcome": "こんにちは{0}、{1}件のメッセージがあります" }
}

Codigo UdonSharp:

// Desde otro script UdonSharp:
textLocalizer.SetParams2("Bender", "5");
// Resultado en español: "Hola Bender, tienes 5 mensajes"
// Resultado en japones: "こんにちはBender、5件のメッセージがあります"

Metodos disponibles para parametros:

  • SetParams(string p0) — reemplaza {0}
  • SetParams2(string p0, string p1) — reemplaza {0} y {1}
  • SetParams3(string p0, string p1, string p2) — reemplaza {0}, {1} y {2}
⚠ En UdonSharp no puedes usar params ni arrays de longitud variable como parametro de metodo. Por eso existen SetParams, SetParams2 y SetParams3 como metodos separados.

Rich Text

Usa richTextFormat para envolver la traduccion en etiquetas de formato. El placeholder {t} representa la traduccion: 

richTextFormat = "<b><color=#00FF00>{t}</color></b>"

// Con key "btn_start" = "Iniciar":
// Resultado: "<b><color=#00FF00>Iniciar</color></b>"
ⓘ Usa siempre {t} como placeholder en richTextFormat. Tambien acepta {0}, pero solo si no tienes parametros dinamicos activos. {t} nunca entra en conflicto.

Pluralizacion

El sistema soporta pluralizacion con variantes de clave. Usa GetPluralValue() desde el LocalizationManager: 

// JSON:
{
  "en": {
    "messages_zero": "No messages",
    "messages_one": "{n} message",
    "messages_other": "{n} messages"
  },
  "es": {
    "messages_zero": "Sin mensajes",
    "messages_one": "{n} mensaje",
    "messages_other": "{n} mensajes"
  }
}

// Codigo:
string text = manager.GetPluralValue("messages", 0);  // "Sin mensajes"
string text = manager.GetPluralValue("messages", 1);  // "1 mensaje"
string text = manager.GetPluralValue("messages", 5);  // "5 mensajes"

Variantes soportadas:

  • {key}_zero — cuando count es 0
  • {key}_one — cuando count es 1
  • {key}_other — para todos los demas valores

Metodos Publicos

MetodoRetornoDescripcion
UpdateText() void Actualiza el texto con la traduccion actual. Llamado automaticamente por el manager.
SetTranslationKey(string newKey) void Cambia la clave de traduccion en runtime y actualiza el texto inmediatamente.
SetParams(string p0) void Establece el parametro {0} y actualiza el texto.
SetParams2(string p0, string p1) void Establece {0} y {1} y actualiza el texto.
SetParams3(string p0, string p1, string p2) void Establece {0}, {1} y {2} y actualiza el texto.
GetTranslationKey() string Devuelve la clave de traduccion actual.
GetManager() LocalizationManager Devuelve la referencia al manager.
SetManager(LocalizationManager m) void Establece el manager manualmente.

Ejemplo completo

Imagina que quieres un texto que muestre "Jugadores: 5/20" y se traduzca:

  1. En el JSON, añade la clave con parametros: 
    "es": { "players_count": "Jugadores: {0}/{1}" }
"en": { "players_count": "Players: {0}/{1}" }
"ja": { "players_count": "プレイヤー: {0}/{1}" }
  2. Añade un componente TextLocalizer al texto, con translationKey = "players_count"
  3. Desde tu script UdonSharp, actualiza los parametros: 
    textLocalizer.SetParams2(
    currentPlayers.ToString(),
    maxPlayers.ToString()
);
  4. Resultado segun idioma:
    • Español: "Jugadores: 5/20"
    • English: "Players: 5/20"
    • Japanese: "プレイヤー: 5/20"
